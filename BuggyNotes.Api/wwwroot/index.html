                ⓘ What is this?
                

                    Demonstrates performance and security aspects of cryptography & hashing.
                    
    
                        
  • AES-GCM (safe) – confidentiality + integrity (auth tag).
    • 
                        
  • AES-CBC (insecure) – fixed IV, no integrity → vulnerable.
    • 
                        
  • PBKDF2 (safe) – slow, salted password hashing.
    • 
                        
  • SHA-256 (insecure) – fast, unsalted; unsuitable for passwords.
